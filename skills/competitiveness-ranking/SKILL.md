# 活水竞争力排行榜

## 技能信息

```yaml
name: 活水竞争力排行榜
description: >
  量化员工在内部活水市场中的综合竞争力，并生成部门/个人多维度排行榜，
  让各部门人才活水健康度变得可感知、可对比。基于"活跃度×成功率×影响力"三维计分模型，
  自动接入员工历史活水行为数据，支持按周期（月/季/年）动态刷新排行。
module: 人才竞争态势感知
trigger: 员工或管理者进入"竞争力排行"页面时自动计算并展示；也可作为管理驾驶舱的常态化监控看板
boundary: 只做"数据量化+排行展示"，不涉及员工潜力预测、发展路径规划或岗位匹配建议
```

---

## 核心机制

### 三维计分模型

| 维度 | 指标 | 数据来源 | 权重 |
|------|------|----------|------|
| **活跃度 (A)** | 活水申请频次、浏览岗位次数 | 活水系统行为日志 | 30% |
| **成功率 (S)** | 面试通过率、转岗成功率 | HR 面试系统、转岗审批记录 | 40% |
| **影响力 (I)** | 转岗后绩效、新岗位留存时长 | 绩效系统、在职时长记录 | 30% |

### 个人竞争力指数 (PCI)

```
PCI = (A_score × 0.30) + (S_score × 0.40) + (I_score × 0.30)
```

- **A_score**: 标准化后的活跃度得分（0-100）
- **S_score**: 标准化后的成功率得分（0-100）
- **I_score**: 标准化后的影响力得分（0-100）

### 部门健康指数 (DHI)

```
DHI = 部门内员工 PCI 均值 × 部门活水参与率 × 部门人才留存率
```

## 数据接入

| 数据类型 | 字段 | 来源系统 |
|----------|------|----------|
| 活水申请记录 | 申请时间、目标岗位、申请状态 | 活水平台 |
| 面试记录 | 面试结果、面试官评价 | 招聘/面试系统 |
| 转岗记录 | 转岗时间、原部门、新部门 | OA/HR系统 |
| 绩效数据 | 转岗后绩效评级 | 绩效管理系统 |
| 在职时长 | 入职时间、转岗后在职时长 | 人事系统 |

---

## 排行榜维度

### 1. 个人排行榜
- **全公司排行**: 所有员工 PCI 排名
- **部门内排行**: 同部门员工 PCI 排名
- **岗位序列排行**: 同岗位序列员工 PCI 排名

### 2. 部门排行榜
- **全公司部门排行**: 所有部门 DHI 排名
- **一级部门排行**: 按一级部门聚合 DHI
- **二级部门排行**: 按二级部门聚合 DHI

### 3. 周期刷新
- **月度排行**: 每月 1 日 00:00 自动刷新
- **季度排行**: 每季度首月 1 日 00:00 自动刷新
- **年度排行**: 每年 1 月 1 日 00:00 自动刷新

## 使用示例

### 场景 1: 员工查看个人竞争力
```
用户: 查看我的活水竞争力
系统: 
  - 计算该员工 PCI（活跃度 78 / 成功率 85 / 影响力 72 → PCI = 79.1）
  - 展示三维雷达图
  - 显示全公司排名: 第 156/3200 名（前 5%）
  - 显示部门内排名: 第 3/45 名
  - 显示同岗位序列排名: 第 8/120 名
```

### 场景 2: 管理者查看部门健康度
```
用户: 查看技术部活水健康度
系统:
  - 计算技术部 DHI = 72.5
  - 全公司部门排名: 第 5/20 名
  - 部门内员工 PCI 分布柱状图
  - 近 6 个月 DHI 趋势图
  - 活水参与率: 68%（公司均值 55%）
```

### 场景 3: 周期排行榜刷新
```
系统定时任务:
  - 每月 1 日 00:00 触发月度排行计算
  - 聚合上月活水行为数据
  - 更新所有员工 PCI 及部门 DHI
  - 推送排行榜更新通知至管理驾驶舱
```
